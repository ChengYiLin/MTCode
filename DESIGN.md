## Overview

Hahow is Taiwan's largest online learning marketplace — a platform where creators sell skills and learners take their first steps into something new. The visual language reflects that mission: **approachable, clear, and energizing without being loud**.

The base canvas is pure white (`{colors.canvas}`) holding cool-grey body text (`{colors.body}` — #595959). The single brand voltage is **Hahow Teal** (`{colors.primary}` — #00ccb4), carried by every primary CTA, interactive link, progress fill, and focus ring. It reads as modern and friendly — not clinical, not aggressive.

**Amber** (`{colors.secondary}` — #ffb21a) has one job: the course rating star. It appears nowhere else as a system action color. **Orange** (`{colors.promotion}` — #fa8c16) is the sale and pricing color — it signals commercial action (discounts, cart CTAs, limited-time offers) and never bleeds into navigation or neutral UI.

No custom typeface. The system font stack (`system-ui, -apple-system, 'Helvetica Neue'`) adapts to the learner's OS: clean and legible in Traditional Chinese, Japanese, and Latin. Weight pairing is two-speed: **600 for titles, 400 for body**. No intermediate weights, no decorative cuts.

**Key Characteristics:**
- White canvas. Interactive teal (`{colors.primary}`) is the single primary brand voltage.
- Amber reserved exclusively for star ratings. Orange reserved for pricing/promotion.
- Two font weights only: 600 bold and 400 regular.
- Responsive typography: titles scale down one step on screens narrower than 768px.
- Soft radii (4px buttons, 8px cards, 16px panels) — human and warm, never pill-shaped.
- Layered elevation via shadows (levels 10–50), not tinted surfaces.
- Teal gradient (`{colors.gradient-down-left}` and variants) for hero bands and decorative sections.

### Principles
- **Two weights only: 600 and 400.** No 500, no 700.

- Max content width: ~1200px.

### Whitespace Philosophy
Generous editorial pacing with clean white canvas. Cards carry their own padding; section bands use 64px vertical rhythm. Learner's content (course cover images, creator avatars) is given breathing room — UI chrome recedes.

## Elevation & Depth

Depth is expressed exclusively via box-shadow. No tinted surface layers; cards float above canvas via shadow, not color lift.

| Level | Token | Shadow | Use |
|---|---|---|---|
| Flat | — | none | Body bands, form fields |
| Low | `shadow[10]` | 0 2px 1px rgba(0,0,0,.05) + 0 0 1px rgba(0,0,0,.25) | Text block floating backgrounds |
| Card | `shadow[20]` | 0 0 2px rgba(0,0,0,.04) + 0 4px 10px rgba(0,0,0,.08) | Course cards, dropdowns |
| Popover | `shadow[30]` | 0 0 8px rgba(0,0,0,.04) + 0 6px 20px rgba(0,0,0,.12) | Tooltips, popovers, menus |
| Drawer | `shadow[40]` | 0 0 8px rgba(0,0,0,.08) + 0 6px 24px rgba(0,0,0,.16) | Side drawers, sheets |
| Modal | `shadow[50]` | 0 16px 32px rgba(0,0,0,.16) + 0 0 8px rgba(0,0,0,.08) | Modal dialogs, highest elevation |

## Do's and Don'ts

### Do
- Reserve `{colors.primary}` (Teal) for interactive CTAs, links, focus rings, and progress indicators.
- Use `{colors.promotion}` (Orange) only for sale prices, discount badges, and purchase-action buttons.
- Keep `{colors.secondary}` (Amber) exclusively on rating stars.
- Use the two-weight system: 600 for titles and emphasis, 400 for body.
- Apply `theme.style.typography.*` for standard typography — it handles responsive sizing automatically.
- Use shadow levels 10–50 for elevation. Never fake depth with tinted backgrounds.
- Keep course card images to 16:9 (rectangle) or 1:1 (square) aspect ratios.

### Don't
- Don't use `{colors.promotion}` (Orange) for non-commercial interactive elements. Navigation, tabs, and focus rings must use teal.
- Don't use `{colors.secondary}` (Amber) as a CTA or text color. Its sole role is the star rating.
- Don't introduce a third font weight. 500 (medium) is not part of the system — it blurs the title/body distinction.
- Don't manually specify letter-spacing on body or caption text. Tight tracking harms CJK readability.
- Don't add border-radius beyond `{rounded.l}` (16px). Pill shapes are outside the token set.
- Don't use the teal gradient on small UI elements (buttons, tags). Gradients are reserved for full-width hero and enrollment bands.
- Don't hardcode hex colors in components — always reference `theme.sys.color.*` tokens in code.

## Responsive Behavior

### Breakpoints

| Name | Width | Key Changes |
|---|---|---|
| Mobile | < 768px | Title scale steps down one level; course grid 1-up; horizontal cards collapse to vertical; nav hamburger. |
| Tablet | 768–1024px | Course grid 2-up; standard type scale resumes; full navigation. |
| Desktop | ≥ 1024px | Course grid 4-up; full layout. |
| Wide | > 1200px | Content caps at 1200px. |

### Typography Responsive Rule
`theme.style.typography.*` automatically applies the small scale (< 768px) and large scale (≥ 768px). Do not override with manual breakpoints unless you need a non-standard intermediate size — in that case, use `theme.utils.convertTypographyStyle()`.

### Touch Targets
- Primary buttons: 40px height minimum (WCAG AA). Large variant at 48px (AAA).
- Icon buttons: 40×40px tap target minimum.

### Collapsing Strategy
- Top nav collapses to hamburger below 768px.
- Course card grid: 4-up → 2-up → 1-up (vertical card).
- Horizontal card (`course-card-horizontal`): splits at 768px, collapses to vertical stack below.
- Modal: full-screen bottom sheet on mobile.

## Iteration Guide

1. One component at a time. Changes to token values cascade — verify affected surfaces in Storybook before shipping.
2. Buttons: `{rounded.s}` (4px). Cards: `{rounded.m}` (8px). Panels: `{rounded.l}` (16px).
3. New variants live as separate entries in `components:` and separate styled-component variants in code.
4. Use `{token.refs}` in this file — never inline hex.
5. Promotion/sale surfaces are the only place `{colors.promotion}` appears. New commercial UI surfaces follow this rule.
6. Amber stays on ratings. If a design uses amber elsewhere, that is a deviation — flag it.
7. New icon components inherit size from `font-size` and color from `color`. Do not hardcode dimensions inside the icon component.
8. Browse all available icons: `npm run storybook:design-system` → HHIcon Overview.
